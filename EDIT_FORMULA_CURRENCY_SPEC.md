# Edit Formula 货币跟随规格（建议）

## 目标行为

**在设置好 Currency 后，点击 Edit，Edit Formula 弹窗里的 Currency 应跟随「已设置好的货币」。**

- 用户在 Data Capture Summary 某一行上已选择的货币（例如 JPY），应视为「该行设置好的货币」。
- 当用户点击该行的 Edit（编辑公式）时，Edit Formula 弹窗内的 **Currency** 下拉应**默认显示该行已设置的货币**（如 JPY），而不是被覆盖为 MYR 或其他默认值。

---

## 这样设计的好处

1. **符合用户预期**  
   行上已经选过 JPY，编辑时弹窗里理应还是 JPY，避免「一打开就变成 MYR」的困惑。

2. **减少误操作**  
   若每次打开 Edit 都变成 MYR，用户必须记得再选一次 JPY，容易漏选或选错。

3. **与「行数据即真相」一致**  
   行的 Currency 列（如 (JPY)）是已保存/已展示的真相，Edit Formula 只是编辑该行，应优先以该行的货币为准。

4. **实现点清晰**  
   逻辑可以归纳为一条规则：**打开 Edit Formula 时，若当前是编辑已有行，则弹窗的 Currency 以「当前编辑行」的 Currency 为准。** 不依赖别处是否「事先把 1SLOT 设成 MYR」等难以排查的配置。

---

## 建议的规则（规格级，不涉及具体代码）

| 场景 | 期望行为 |
|------|----------|
| 用户点击**某一行**的 Edit（该行 Currency 列已显示如 JPY） | Edit Formula 弹窗里的 Currency 默认选中 **JPY**（即与该行一致） |
| 用户在 Edit Formula 弹窗里**改选**其他账户 | Currency 按新账户的可用货币重新加载；若新账户支持当前行货币则可保留，否则按业务规则选默认（如 MYR 或第一项） |
| 用户是**新增**一行（不是编辑已有行） | 不强制「跟随行上货币」；可按账户默认或现有逻辑处理 |

**核心一条：**  
「设置好 Currency」= 该行在 Summary 表上的 Currency 列已有值（如 JPY）。  
「点击 Edit」= 打开 Edit Formula。  
「Edit Formula 里的 Currency 跟着设置好的货币」= 弹窗内 Currency 的初始值 = 该行的 Currency（JPY）。

---

## 与现有问题的关系

- 当前现象：点击 Edit 后弹窗里 Currency 常变成 MYR。  
- 本规格要达成的效果：点击 Edit 后弹窗里 Currency **保持为该行已设置的货币**（如 JPY）。  
- 实现时需保证：**凡是会决定「弹窗打开时 Currency 显示什么」的逻辑**（例如加载账户货币、恢复上次选择等），都要在「编辑已有行」的情况下，**优先采用当前编辑行的 Currency**，而不是无条件的 MYR 或其它默认。

---

## 小结

采用「设置好 Currency 后，点击 Edit，Edit Formula 里的 Currency 跟着设置好的货币」这一条规则，会更好，因为：

- 行为明确、易理解；
- 避免被默认 MYR 覆盖造成的困惑；
- 实现目标单一：以当前编辑行的 Currency 为弹窗初始值即可。
